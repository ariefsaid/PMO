## 1. Decisions (one at a time)

### D1 — Mount point: **third view on the existing Tasks-tab `ViewToggle`**
`TasksTab.tsx` already owns a `ViewToggle<ViewMode>` with `list | board`. v1 adds a
third option `timeline`. This is the cleanest mount: the tab already loads BOTH reads we
need (`useTasks` + `useMilestones`), already has the empty/loading/error gating, and a
view-toggle is the established "same data, different lens" pattern here. **No new tab, no
new sub-route** — avoids touching `ProjectDetail.tsx`'s `PTab` union / routing / default-tab
logic (`tabFromParam`), so the change is contained.

- `ViewMode` becomes `'list' | 'board' | 'timeline'`.
- The toggle gains `{ value: 'timeline', label: 'Timeline' }`.
- A new branch: `state === undefined && view === 'timeline'` renders `<ProjectGantt tasks={all} milestones={milestoneList} />`.

### D2 — Data path: **reuse, confirmed — zero new data layer**
`TasksTab` already calls `useTasks(projectId)` (→ `all: TaskWithRefs[]`, carrying
`start_date` / `end_date`) and `useMilestones(projectId)` (→ `milestoneList:
MilestoneWithProgress[]`, carrying `target_date` / `sort_order` / `name`). `ProjectGantt`
is a **pure presentational** component fed those two already-fetched arrays as props — it
fires **no** hook, query, RPC, RLS, or migration. RLS already scopes both reads.

### D3 — Geometry is a **pure util** `src/lib/gantt/ganttLayout.ts` (unit-tested)
All date→x math lives in one deterministic pure module (mirrors `sCurve.ts` /
`monthMatrix.ts`). It takes tasks + milestones + a `todayIso`, returns a normalized
`GanttModel` of positioned bars (0..1 fractions) + milestone markers + today fraction +
the axis tick dates. `ProjectGantt` only maps fractions → `%` left/width style and paints
tokens. Local-date parsing via the SAME approach as `monthMatrix.parseLocalDate` (no UTC
shift) — re-exported, not re-implemented (D7).

### D4 — Dependency arrows: **DEFER to v2 (no connector lines in v1)**
The `task_dependencies` edges ARE present on `TaskWithRefs.dependencies[]` (seeded table),
so the data is in hand. But arrow/connector routing between time-positioned bars across a
two-level milestone-grouped layout (vertical lane assignment, elbow routing, scroll-sync,
overlap avoidance) is **not** "trivially cheap" — it needs its own geometry pass and a11y
story. v1 ships **task bars + milestone markers + today line only**. v2 adds arrows.
v1 DOES surface dependency *presence* cheaply: a bar with `dependencies.length > 0` gets a
small "depends on N" affordance in its row label / title (text, not a drawn arrow) so the
information is not lost — see Task 6 (AC-GANTT-008).

### D5 — Undated tasks: **grouped honestly, not crashed**
- Task with BOTH `start_date` and `end_date` → a positioned **bar**.
- Task with exactly ONE of start/end → a single-day **point/flag** at the present date
  (zero-width-bar rendered as a diamond), tooltip "start only" / "due only".
- Task with NEITHER → excluded from the plotted lanes and listed in an **"Undated (N)"**
  footer row so it is acknowledged, never silently dropped. (AC-GANTT-005)

### D6 — Two-level grouping mirrors the list view
If milestones exist, bars are grouped under milestone lanes ordered by `sort_order`
(then name), with an **Ungrouped** lane trailing — identical ordering to
`MilestoneGroupedList`. Each milestone lane header shows its `target_date` marker on the
axis. No milestones → a single flat lane. (AC-GANTT-006)

### D7 — Reuse `parseLocalDate` from `monthMatrix.ts`
Re-export `parseLocalDate` (and add a sibling `toIso` export there if needed) rather than
re-implementing date parsing — single source of truth for "calendar date, no TZ shift".

### D8 — Mobile @390: horizontal-scroll, contained (CSS-only; NO new hook needed)
The Gantt is a single horizontally-scrollable lane block (`overflow-x-auto`), min-width
sized so bars never crush below ~`640px`, wrapped so it never causes **page** overflow
(the wrapper is `max-w-full overflow-hidden` around an inner `overflow-x-auto`). The
kanban `useKanbanMobileScroll` stage-indicator hook is for *discrete columns*; a Gantt is a
*continuous* axis, so its nearest-column model does not apply — we reuse the **visual**
scroll-fade/contained-scroll PATTERN (the `overflow-x-auto` + edge-fade styling), not the
hook. A scroll-fade edge is applied with the existing fade utility classes. Today line +
axis month ticks scroll WITH the bars (same positioned track), so alignment holds.

### D9 — Tokens / a11y
Strictly DESIGN.md tokens: bars use the One-Blue series token surfaces already used
elsewhere (`bg-primary/15` track-fill family, `border-border`, `bg-card`, `text-muted-foreground`,
`text-[11.5px]/[13px]` type scale, 32px control heights on the toggle — inherited from
`ViewToggle`). Status is encoded by the task's existing `STATUS_PILL` variant tint **plus a
text label in the bar / its title** — never color-only. The whole figure is wrapped in
`role="img"` with an `aria-label` summary (mirrors `ProjectSCurve`'s `<figure>` pattern);
the today marker carries `aria-hidden` + a visible "Today" caption. Respects
`prefers-reduced-motion` (no animated bar grow when set).

### D10 — ADR: **none required**
This follows the established S-curve / calendar precedent (pure util + presentational
component, no schema, no auth surface). No architectural/irreversible/cross-cutting
decision. If review disagrees, number it `0023-read-only-gantt.md` after the current ADRs.

### `buildGanttModel` contract (load-bearing signatures)

```ts
// src/lib/gantt/ganttLayout.ts
import type { TaskWithRefs } from '@/src/lib/db/tasks';
import type { MilestoneWithProgress } from '@/src/lib/db/milestones';

/** A task drawn on the axis: fraction left/width in [0,1] of the model's date span. */
export interface GanttBar {
  id: string;
  name: string;
  status: TaskWithRefs['status'];
  /** Fraction (0..1) of the span where the bar starts. */
  left: number;
  /** Fraction (0..1) width. A point/flag (one-sided date) has width 0. */
  width: number;
  /** 'bar' (both dates) | 'point' (exactly one date). */
  kind: 'bar' | 'point';
  startIso: string | null;
  endIso: string | null;
  /** Count of outgoing dependency edges (v1 surfaces presence as text, D4). */
  dependsOnCount: number;
}

/** A milestone target-date marker positioned on the axis. */
export interface GanttMarker {
  id: string;
  name: string;
  targetIso: string;
  left: number; // fraction 0..1
}

/** One lane = a milestone (or the trailing Ungrouped lane) holding its bars. */
export interface GanttLane {
  milestoneId: string | null;
  label: string;            // milestone name | 'Ungrouped'
  marker: GanttMarker | null; // the lane's own target-date marker, if dated
  bars: GanttBar[];
}

export interface GanttAxisTick { iso: string; left: number; label: string; }

export interface GanttModel {
  /** Inclusive ISO date span [min,max] that defines the 0..1 axis, or null if nothing dated. */
  span: { startIso: string; endIso: string } | null;
  lanes: GanttLane[];
  /** Today's fraction (0..1) if within span, else null (don't draw the line). */
  todayLeft: number | null;
  /** Month-boundary ticks for the axis. */
  ticks: GanttAxisTick[];
  /** Tasks with NEITHER date — acknowledged in a footer, not plotted (D5). */
  undated: { id: string; name: string }[];
  /** True when there is nothing dated to plot at all (caller → empty state). */
  isEmpty: boolean;
}

export function buildGanttModel(
  tasks: TaskWithRefs[],
  milestones: MilestoneWithProgress[],
  todayIso: string,
): GanttModel;
```

**Span rule:** `span.startIso` = min over all present task `start_date`/`end_date` AND
milestone `target_date`; `span.endIso` = max of the same set. If start===end, pad the span
by ±1 day so a single-date project still has nonzero width (`width 0` bars still render as a
point). `left = clamp01(daysBetween(span.start, d) / spanDays)`. Pure, deterministic,
local-date parsing.

## 3. Requirements (EARS)

- **FR-GANTT-001** — Where the Tasks tab is shown, the system shall offer a **Timeline** view alongside List and Board.
- **FR-GANTT-002** — When the Timeline view is selected, the system shall render each task that has both a start and end date as a horizontal **bar** positioned and sized by its date range on a shared time axis.
- **FR-GANTT-003** — When the Timeline view is selected, the system shall render each dated **milestone** as a target-date **marker** on the same time axis.
- **FR-GANTT-004** — While today falls within the plotted date span, the system shall render a **today** reference line on the axis.
- **FR-GANTT-005** — Where a task has exactly one of start/end date, the system shall render it as a single-point **flag** (not a bar); where a task has neither, the system shall list it in an **"Undated"** footer and shall not plot it.
- **FR-GANTT-006** — Where milestones exist, the system shall **group** bars into per-milestone lanes ordered by `sort_order` then name, with an Ungrouped lane last.
- **FR-GANTT-007** — Where no task and no milestone carries any date, the system shall show an honest **empty** state and shall not crash.
- **FR-GANTT-008** — The Timeline view shall be **read-only**: it shall expose no drag, reschedule, edit, or write affordance.
- **NFR-GANTT-PERF-001** — The Timeline view shall add **no** network round-trip — it consumes the Tasks tab's already-fetched `useTasks` + `useMilestones` caches.
- **NFR-GANTT-A11Y-001** — Bar status shall be conveyed by label/text in addition to tint (never color-only); the figure shall carry a `role="img"` summary `aria-label`; motion shall respect `prefers-reduced-motion`.
- **NFR-GANTT-RESP-001** — At 390px the Timeline shall scroll horizontally within a contained region and shall not cause page-level horizontal overflow.
