## 0. Design brief (one decision at a time)

### 0.1 W2-1 — the keystone: budget must be DERIVED everywhere it is money

The shipped truth (migration 0032, pgTAP 0074): `projects.spent` is a stored column **no trigger populates** → it was a silent "$0" money bug, fixed by deriving spent from procurements. **`projects.budget` has the identical disease** — it is a stored column never synced to the Active budget-version line-item total. The canonical derived basis already exists and is already used by the **pipeline lens** of `get_executive_dashboard` (0027/0032):

```sql
coalesce((select sum(li.budgeted_amount)
          from budget_versions v join budget_line_items li on li.budget_version_id = v.id
          where v.project_id = p.id and v.status = 'Active'), 0)
```

The fix makes that subquery the budget basis **everywhere budget is consumed as money or at-risk**, not just the pipeline lens:

1. **`get_executive_dashboard`** (current authority = `0032_fix_top_projects_spent.sql`, which superseded 0009/0027): the `active_committed` CTE's `p.budget` (feeds `projects_at_risk`) and the `top_projects` select's `p.budget` both become the derived Σ. A line-item-budgeted project whose stored `projects.budget=0` is currently **excluded** from `projects_at_risk` by the `budget > 0` guard and shows "$X of $0" — after the fix it is correctly counted and its util% is real.
2. **`get_projects_delivery`** (`0026:40,47,59`): `p.budget` in the `committed` CTE → same derived Σ; the FE Projects "Budget used" + at-risk cells read this.
3. **FE OverviewTab** (`OverviewTab.tsx:64,66,69,79`): replace `project.budget ?? 0` with the derived budget from `useProjectBudget(project.id)` (already imported as a hook elsewhere; `deriveProjectBudget` DAL = Σ Active line-items) for the util%/at-risk gate. The Budget-snapshot card already uses `activeSnapshot(budgetVersions, …)` (derived) — only the top "Budget utilization" card + the `isAtRiskByCommitted` gate read the stored column.

**Decision — basis subquery is copied verbatim, not abstracted into a SQL function.** 0032 already inlines the spent subquery 3×; the budget Σ is already inlined in the pipeline lens. Matching that local idiom keeps the diff reviewable and avoids a new shared SQL function (no ADR needed). Reversible via `supabase db reset`; `security invoker` + `search_path=public` + the anon revoke are preserved verbatim from 0032.

**Owning layer (ADR-0010):** the RPC behavior change is owned by **pgTAP 0075** (`AC-W2-1-RPC-01..03`). The FE gate is owned by a **Vitest guard** (`AC-W2-1-FE-01`). Mirror `0074_top_projects_spent_from_procurements.test.sql` exactly (isolated UUID prefix `00750000-…`, run as a test-org Executive).

### 0.2 W2-3 — date-only UTC off-by-one (1 root cause)
Canonical guards already exist: `formatDate` (`src/lib/format.ts:51` — date-only parsed at local midnight) and `parseLocalDate` (`src/lib/calendar/monthMatrix.ts:28`) + `toIso` (`:34` — local `YYYY-MM-DD`). The fix deletes the bypassing local helpers and routes through these. **MyTasks already uses `formatDate`** (lines 152-154) — it is NOT a W2-3 site; the audit cites only ProjectDetailHeader, OverviewTab, toWorkbookBuffer, and the win-rate boundary.

### 0.3 W2-4 — false-empty/false-zero on fetch error (1 root cause)
Every one of the 6 hooks is a standard `useQuery` → `isError`/`refetch` are already available; the components just never read them. The shared error affordance is `<ListState variant="error" onRetry={refetch}/>` (already the app idiom — `RelatedList`, `TasksTab`, `MyTasks`, `OverviewTab` all use it). The KPITile (a leaf metric tile) gets a new additive `error?` prop → renders an em-dash `—`. **Shared-first:** the `KPITile` `error` prop (W2-4a) lands before the PMDashboard band consumer (W2-4d).

### 0.4 W2-10 — ViewToggle role accuracy (shared component, additive)
`ViewToggle` hardcodes `role="tablist"`/`role="tab"`/`aria-selected` — correct for tab-panel switchers (e.g. project-detail tabs) but wrong for filter/scope/basis uses that have **no tabpanel** (WinRateCard basis/period, sales scope). Add an additive `role?: 'tablist' | 'group'` prop (default `'tablist'` = zero behavior change for every current caller). When `'group'`, the container is `role="group"` and each option uses `aria-pressed` instead of `aria-selected`/`role="tab"`. Only the WinRateCard non-tab uses are re-pointed in this wave (the minimum the audit cites); other filter uses are backlog-noted, not touched.
