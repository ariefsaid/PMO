## Design brief (impeccable shape)

**1. Feature summary.** The app shell currently renders a browser-style workspace **tab strip** (3rd grid row) backed by a `WorkspaceTabsProvider` + sessionStorage + URL→tab sync. The owner finds it noisy for an app this size. Remove the tab layer entirely; navigation stays via the **rail + route-derived breadcrumb + ⌘K**. Separately, upgrade the **⌘K command palette** from module-only navigation to **record search** (open a specific project / opportunity / PR by name or code) sourced from the already-cached TanStack lists.

**2. Primary user action.** Get to any record or module fast: click the rail (modules), follow the breadcrumb (up the hierarchy), or hit ⌘K and type a record name/code to jump straight to its detail route.

**3. Design direction.** Restrained (product register floor). Identity = RIS "Quiet Control Surface": one `primary` blue for the only interactive affordance, borders-not-shadows, Inter + `tabular-nums`, SF Mono for codes, flat-by-default. Anchor references: Linear ⌘K, Raycast root search, GitHub's `t`-file-finder. No tab metaphor at all (the second-order slop trap "dashboard app → browser tabs" is exactly what we remove).

**4. Scope.** Production-ready. Breadth: the whole shell chrome + the 6 page call-sites that depend on the tab API + the palette. Interactivity: shipped-quality, keyboard-first. Time intent: polish until it ships (it gates merge via `/design-review`).

**5. Layout strategy.** `AppShell` grid drops from 3 rows to 2: `gridTemplateRows: var(--header-h) 1fr` and `gridTemplateAreas: "rail header" "rail main"`. The `tabstrip` slot/prop is removed. Everything else (rail width, header height, main scroll, skip-link, mobile drawer) is unchanged — minimal blast radius. The ⌘K palette gains a **Records** group above **Navigate**, capped + ranked so it never becomes a wall of rows.

**6. Key states (palette).** default (modules + empty query) · typing/filtering · **loading** (record lists still fetching → skeleton rows, not a spinner) · **results** (grouped Records/Navigate) · **no-results** (calm message, existing pattern) · **error** (a record list failed to load → inline note + module nav still works, never a blank palette) · edge: query matches a code exactly (rank it first), 200+ records (cap to 8 per group), duplicate names across modules (mono code + module sub-label disambiguate).

**7. Interaction model.** ⌘K / Ctrl-K opens (unchanged). Typing filters across cached records + modules with a **120 ms debounce**. ArrowUp/Down move the roving selection across the **flat** result order (Records then Navigate); Enter runs the selected item → `navigate(detailPath)` and closes; Esc closes and restores focus to the trigger. Clicking a row does the same. Opening a record from a list page / kanban / detail-page hydration is now a plain `navigate()` — no tab is created.

**8. Content requirements.** Palette input placeholder stays `Search projects, PRs, customers, or run a command…` (already on-brand, verb-led). Records group label `Records`; nav group label `Navigate`. Record row: title (Inter 14/500) + module sub-label (`muted-foreground` 11.5px) + mono code badge (SF Mono 11px). Loading skeleton rows reuse the row geometry. No emoji anywhere (existing `Icon` set only).

**9. Open questions.** See bottom; none block the build.

## Architecture decision: how the tab API collapses

`useWorkspaceTabs()` is consumed by **6 page components** purely for three things:
- `openRecord(tab)` → navigate to a detail route **and** register/refocus a tab.
- `openModule(key)` → navigate to a module index **and** register/refocus a tab (also used as "go back").
- `setDirty(id, bool)` → flip the amber dirty dot on a record tab.

With tabs gone, each collapses to a plain `react-router` `navigate()` (and `setDirty` becomes a no-op that we delete). The breadcrumb, which today reads `ws.tabs`/`ws.activeId`, is re-derived **from the route** via a new pure helper `breadcrumbForPath(pathname, recordLabel?)`. This keeps "URL is the source of truth" (the existing invariant) while dropping the entire tab-state machine.

**Net deletions:** `TabStrip.tsx`, `WorkspaceTabsProvider.tsx`, `workspaceTabs.ts`, `routeMatch.ts` (tab-shaped parts), and their 4 test files (`TabStrip.test`, `WorkspaceTabsProvider.test`, `tabRouteSync.test`, plus the tab-specific cases in `routeMatch`). The `--tabstrip-h` CSS var and the `tabstrip-scroll` style are removed.

**Net additions:** a small `nav.ts` (or extend `routeMatch.ts` renamed conceptually) exporting `MODULES` (kept — the rail + palette need the module list) and `breadcrumbForPath()`; a `useRecordSearch()` hook that indexes the three cached lists into `PaletteItem[]`.

- [ ] `nav-state-active`: rail active item keeps `bg-primary/10 + text-primary + font-semibold + aria-current` (existing) — preserved.
